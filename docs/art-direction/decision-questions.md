# Decision Questions

These are useful questions for Steve before turning the art direction into production UI. They should not block small prototypes.

## World Framing

- Should the command station feel like a ship bridge, a planetary command bunker, an orbital station, or an abstract strategy console?
- Should the commander feed use human officers, alien race advisors, an AI system voice, or a mix?
- Should each race eventually have its own advisor/console skin, or should the command interface stay mostly faction-neutral?

## Layout

- Should the main game target a fixed 16:9 frame with letterboxing, or should it adapt more aggressively to ultrawide and mobile layouts?
- Which panel should be visually dominant after the map: commander feed, resource console, action console, or event log?
- Should chat remain a modern utility panel, or should it be restyled as an in-world communications terminal?

## Motion And Tone

- How intense should alert states be for black holes, asteroid damage, and battle losses?
- Should routine updates be calm and quiet, or should the interface feel constantly active?
- Should the commander feed eventually include optional voice/TTS, or stay visual/text-only?

## Production Asset Style

- Should final assets lean more painterly, pixel/dithered, or clean pre-rendered?
- Should generated concept art be used only for reference, or can generated assets be refined into production placeholders when clearly labeled?
